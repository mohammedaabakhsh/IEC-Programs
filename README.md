# IEC-Programs — نظام إدارة برامج وورش مركز الابتكار وريادة الأعمال

تطبيق ويب لإدارة الورش والبرامج: تضيف ورشة من داخل التطبيق، يتولّد لها تلقائيًا رابط تقييم (Google Form) ورمز QR خاص فيها، وتشوف نتائجها (عدد الردود، متوسط الرضا، تقييم المدرب والتنظيم، والملاحظات) في صفحة خاصة فيها، محفوظة دائمًا ترجع لها وقت ما تبي.

بدون سيرفر مدفوع وبدون قاعدة بيانات منفصلة: الواجهة صفحات ويب ثابتة (GitHub Pages)، والخادم الخلفي Google Apps Script Web App مجاني، والتخزين في ملف Google Sheets واحد. التقييم نفسه يتم عبر Google Form (نموذج واحد مشترك، يتغيّر عنوانه تلقائيًا مع كل ورشة جديدة).

## هيكل المشروع

```
IEC-Programs/
├── apps-script/          (الخادم الخلفي - يُنشر داخل Google Sheets)
│   ├── Config.gs           إعدادات عامة
│   ├── Setup.gs             إعداد أولي + إنشاء النموذج المشترك
│   ├── Programs.gs          دوال جدول الورش + بناء رابط النموذج
│   ├── Workshops.gs         إنشاء ورشة جديدة (createWorkshop_)
│   ├── Dashboard.gs         حساب الإحصاءات لكل ورشة
│   ├── Api.gs                واجهة API (doGet / doPost)
│   └── appsscript.json
└── docs/                  (التطبيق الأمامي - يُنشر على GitHub Pages)
    ├── index.html           قائمة الورش + زر إضافة ورشة جديدة
    ├── workshop.html         صفحة تفاصيل وإحصاءات ورشة واحدة
    └── assets/
        ├── config.js          رابط الخادم API_URL
        ├── style.css
        ├── app.js             منطق صفحة القائمة والإضافة
        └── workshop.js        منطق صفحة تفاصيل الورشة
```

الواجهة داخل مجلد docs/ (وليس web/) لأن GitHub Pages يدعم فقط مجلد / (root) أو /docs.

## خطوات التركيب والنشر

### 1) الخادم الخلفي (Google Apps Script)

1. أنشئ ملف Google Sheets جديد فارغ (Sheets.new).
2. من القائمة: الإضافات (Extensions) ← Apps Script.
3. احذف محتوى Code.gs الافتراضي، وأنشئ ملفات بنفس أسماء ملفات مجلد apps-script/ هنا (Config.gs, Setup.gs, Programs.gs, Workshops.gs, Dashboard.gs, Api.gs) والصق محتوى كل ملف كما هو.
4. من إعدادات المشروع في محرر Apps Script، فعّل "Show appsscript.json manifest file"، واستبدل محتواه بمحتوى apps-script/appsscript.json.
5. حدّث صفحة الشيت، ثم من قائمة "نظام إدارة الورش" ← "إعداد النظام (مرة واحدة)" ووافق على الصلاحيات (Google تطلب تصريحًا لأن السكربت جديد — اختر حسابك ← Advanced ← Go to (المشروع) unsafe ← Allow). هذا ينشئ ورقتي "الورش" و"استجابات التقييم"، ونموذج Google Form مشترك.
6. في محرر Apps Script: Deploy ← New deployment ← Web app.
   - Execute as: Me
   - Who has access: Anyone
   - اضغط Deploy وانسخ رابط Web App (ينتهي بـ /exec).

### 2) الواجهة الأمامية (GitHub Pages)

1. عدّل الملف docs/assets/config.js وضع رابط Web App الذي نسخته:
   ```
   const APP_CONFIG = { API_URL: 'https://script.google.com/macros/s/XXXXXXXX/exec' };
   ```
2. في مستودع GitHub: Settings ← Pages ← Source: Deploy from a branch ← Branch: main، والمجلد: /docs ← Save.
3. بعد دقيقة أو دقيقتين يظهر رابط التطبيق أعلى نفس الصفحة (Settings ← Pages).

### 3) الاستخدام اليومي

1. افتح رابط التطبيق (GitHub Pages) ← اضغط "إضافة ورشة جديدة" ← عبّي البيانات (الاسم، الوصف، التاريخ، الوقت، المدرب، الفئة المستهدفة، عدد المشاركين، الجهة المنظمة) ← احفظ.
2. يطلع لك فورًا رابط تقييم مخصص للورشة + رمز QR. انسخ الرابط أو صوّر QR وشاركه بنفسك مع المشاركين بعد انتهاء الورشة (بالطريقة اللي تناسبك: واتساب، شاشة، طباعة).
3. عند تعبئة أي مشارك للنموذج، تُحفظ إجابته تلقائيًا في ورقة "استجابات التقييم" بملف Google Sheets.
4. من صفحة الورشة داخل التطبيق (تفتحها من قائمة الورش في الصفحة الرئيسية) تشوف: عدد الردود، المتوسط العام، تقييم المدرب، تقييم التنظيم، الإحصاءات التفصيلية لكل محور، وكل الملاحظات المكتوبة — وتقدر ترجع لها وقت ما تبي لأن كل البيانات محفوظة دائمًا.

## ملاحظة مهمة حول النموذج المشترك

النظام يستخدم نموذج Google Form واحد لكل الورش (بدل إنشاء نموذج منفصل لكل ورشة) لتبسيط الإدارة. عند إنشاء ورشة جديدة، يتغيّر عنوان هذا النموذج تلقائيًا ليعكس اسم آخر ورشة تم إنشاؤها. كل رابط تقييم يحمل معرف الورشة مدمجًا فيه (رابط معبّأ مسبقًا)، فالردود تُنسب دائمًا للورشة الصحيحة بغض النظر عن العنوان الظاهر — لكن إذا أنشأت ورشتين وأرسلت روابطهما في نفس الوقت تقريبًا، سيظهر للمشاركين عنوان آخر ورشة تم إنشاؤها على النموذج (الأسئلة والبيانات المحفوظة تبقى صحيحة ودقيقة لكل ورشة على حدة).

## استكشاف الأخطاء

- "لم يتم ربط رابط الخادم" في الواجهة: لم تعدّل docs/assets/config.js برابط /exec الصحيح.
- خطأ عند إنشاء ورشة (CORS): تأكد أن نشر Web App بصلاحية Anyone، وأن أي تعديل على الكود يُنشر عبر Deploy ← Manage deployments ← تعديل ← نسخة جديدة (الرابط نفسه لا يتغيّر).
- لا يظهر رابط في GitHub Pages: تأكد أن المجلد المختار في Settings ← Pages هو /docs بالضبط، وانتظر دقيقة أو دقيقتين بعد الحفظ.
