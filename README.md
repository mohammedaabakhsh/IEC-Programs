# IEC-Programs — نظام التقييم الإلكتروني الموحّد للبرامج

تطبيق ويب لتوثيق ومتابعة جميع البرامج وورش العمل والجلسات، مع إرسال رابط تقييم تلقائيًا للمشاركين بعد انتهاء كل برنامج، ولوحة متابعة حيّة تعرض مؤشرات رضا المستفيدين.

بدون سيرفر مدفوع وبدون قاعدة بيانات منفصلة: الواجهة صفحات ويب ثابتة (تُستضاف مجانًا على GitHub Pages)، والخادم الخلفي Google Apps Script Web App مجاني يقرأ ويكتب مباشرة في ملف Google Sheets واحد يُستخدم كقاعدة بيانات.

## هيكل المشروع

```
IEC-Programs/
├── apps-script/        (الخادم الخلفي - يُنشر داخل Google Sheets)
│   ├── Config.gs
│   ├── Setup.gs
│   ├── Programs.gs
│   ├── Evaluation.gs
│   ├── Dashboard.gs
│   ├── Api.gs
│   └── appsscript.json
└── docs/                (التطبيق الأمامي - يُنشر على GitHub Pages)
    ├── index.html        لوحة المتابعة
    ├── evaluate.html      نموذج التقييم
    └── assets/
        ├── config.js      رابط الخادم API_URL
        ├── style.css
        ├── dashboard.js
        └── evaluate.js
```

ملاحظة: الواجهة موضوعة داخل مجلد `docs/` وليس `web/` لأن GitHub Pages (خيار Deploy from a branch) يدعم فقط مجلد `/ (root)` أو `/docs`.

## خطوات التركيب والنشر

### 1) الخادم الخلفي (Google Apps Script)

1. أنشئ ملف Google Sheets جديد فارغ (Sheets.new).
2. من القائمة: الإضافات (Extensions) ← Apps Script.
3. احذف محتوى Code.gs الافتراضي، وأنشئ ملفات بنفس أسماء ملفات مجلد apps-script/ هنا (Config.gs, Setup.gs, Programs.gs, Evaluation.gs, Dashboard.gs, Api.gs) والصق محتوى كل ملف كما هو.
4. من إعدادات المشروع في محرر Apps Script، فعّل "Show appsscript.json manifest file"، واستبدل محتواه بمحتوى apps-script/appsscript.json.
5. حدّث صفحة الشيت، ثم من قائمة "نظام التقييم" ← "إعداد النظام (مرة واحدة)" ووافق على الصلاحيات.
6. في محرر Apps Script: Deploy ← New deployment ← Web app.
   - Execute as: Me
   - Who has access: Anyone
   - اضغط Deploy وانسخ رابط Web App (ينتهي بـ /exec).

### 2) الواجهة الأمامية (GitHub Pages)

1. عدّل الملف docs/assets/config.js وضع رابط Web App الذي نسخته:
   ```
   const APP_CONFIG = { API_URL: 'https://script.google.com/macros/s/XXXXXXXX/exec' };
   ```
2. في مستودع GitHub: Settings ← Pages ← Source: Deploy from a branch ← Branch: main، والمجلد: /docs ← Save.
3. بعد دقيقة أو دقيقتين يظهر رابط الموقع أعلى نفس الصفحة (Settings ← Pages)، غالبًا بالشكل: https://mohammedaabakhsh.github.io/IEC-Programs/
4. مهم: ارجع لملف apps-script/Config.gs وعدّل FRONTEND_BASE_URL ليصبح نفس رابط GitHub Pages (بدون / في النهاية)، ثم احفظ الملف في محرر Apps Script (بدون حاجة لإعادة نشر Web App).

### 3) الاستخدام اليومي

1. أضف صفًا جديدًا في ورقة "البرامج" بملف الشيت: اسم البرنامج، النوع، الفئة المستهدفة، الجهة المنظمة، اسم المدرب، تاريخ البدء والانتهاء، عدد المشاركين المستهدف، وبريد المشاركين (مفصول بفواصل).
2. النظام يراقب تاريخ الانتهاء يوميًا (الساعة 8 صباحًا) ويرسل رابط التقييم تلقائيًا لكل مشارك، ويحدّث حالة الإرسال.
3. لإرسال فوري: حدّد صف البرنامج ← قائمة "نظام التقييم" ← "إرسال رابط التقييم للصف المحدد الآن".
4. كل استجابة جديدة تُحفظ في ورقة "استجابات التقييم" وتُحدّث "لوحة المتابعة" فورًا.
5. صفحة index.html تعرض لكل برنامج: عدد الاستجابات، نسبة المشاركة، متوسط الرضا لكل محور، والمتوسط العام، مع رسم بياني.
6. كل أحد الساعة 7 صباحًا يصل بريد تلقائي بملخص لوحة المتابعة. لتغيير المستلم عدّل ADMIN_EMAIL في Config.gs.

## نقاط قابلة للتخصيص لاحقًا

- دورية التقرير الدوري: حاليًا أسبوعي، يمكن تغييرها في ensureTriggers_() بملف Setup.gs.
- نطاق مخصص بدل github.io: يمكن ربطه من إعدادات GitHub Pages.
- صفحة إدارة داخل التطبيق لإضافة برامج بدل تحرير الشيت مباشرة.
- تنبيهات واتساب أو SMS بدل البريد فقط (تتطلب خدمة خارجية مثل Twilio).

## استكشاف الأخطاء

- "لم يتم ربط رابط الخادم" في الواجهة: لم تعدّل docs/assets/config.js برابط /exec الصحيح.
- خطأ عند الإرسال (CORS): تأكد أن نشر Web App بصلاحية Anyone، وأن أي تعديل على الكود يُنشر عبر Deploy ← Manage deployments ← تعديل ← نسخة جديدة (الرابط نفسه لا يتغيّر).
- لا تصل رسائل البريد: تأكد من صلاحية Gmail عند أول تشغيل لـ "إعداد النظام"، وتحقق من مجلد الرسائل المرسلة بحساب Google المرتبط بالشيت.
- لا يظهر رابط في GitHub Pages: تأكد أن المجلد المختار في Settings ← Pages هو /docs بالضبط (وليس /web أو / root)، وانتظر دقيقة أو دقيقتين بعد الحفظ، وحدّث الصفحة.
